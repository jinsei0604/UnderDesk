## プロジェクト概要

**UNDERDESK**（タスクバーの底のダンジョン）: デスクトップ常駐型 放置ダンジョン経営ゲーム。
Godot 4.7 (GDScript)、Steam向け個人開発プロジェクト。

- リポジトリ: `C:\src\underdesk`（OneDrive・日本語パスを避けるため、デスクトップのinvitation-makerとは別）
- リモート: https://github.com/jinsei0604/UnderDesk.git（private）。コミットのたびに `git push origin main` する運用
- Godotポータブル版: `C:\src\tools\godot\Godot_v4.7-stable_win64.exe`（`_console.exe`もあり）
- **企画書（仕様の正）**: `docs/design.md` にリポジトリ内保存済み。§12（コーディング規約）と§5（システム詳細）を優先
- **開発ガイド（最重要）**: リポジトリ直下の `CLAUDE.md` に、アーキテクチャ・鉄則・コンテンツ追加レシピ・未実装機能の設計図が書いてある。**新セッションはまずこれを読むこと**
- デスクトップにショートカット `UNDERDESK.lnk` あり（ゲームを直接起動）

## 現在の到達点（2026-07-12時点、コミット33件、テスト84件全合格）

### コアシステム
- tick制シミュレーション（2秒/tick）、決定的（乱数はUDSim._rngのみ）、オフライン一括計算とリアルタイムが等価（テストで保証）
- グリッド60×8、掘り進むと自動拡張。地層4層（表土/岩盤/地下水脈/古代遺構）
- **自動掘削は蛇行トンネル方式**: 右へ掘り進み、行末で1段下りて左へ折り返す（下に掘るのではなく横穴を掘る見た目。ユーザー要望で変更）
- セーブ: JSON、世代バックアップ3つ、バージョンv4（移行処理あり）

### パーティ制（重要な仕様変更）
- 当初は「手下」が複数いる経営ゲームだったが、**プラン変更で主人公1人が発掘するスタイルに変更**
- 主人公が1人でスタート、**文書の発見数に応じて仲間が最大4人加入**（data/companions/、パーティ上限5人）
- 現在定義済みなのは **companion_2 = リコ（文書5編で加入）のみ**。他3枠は削除済み（プレースホルダのため）
- 仲間はショップや部屋では増やせない（宿舎はフレーバーのみ、雇用は廃止済み）

### 経済
- 掘った資源は**その場で回収袋（pending_loot）へ**（拠点への運搬は廃止、ユーザー要望）
- 大画面を開いた瞬間に一括換金（土1/石2/鉱石5/魔石10 → コイン）、「回収 +Nコイン」表示
- 金塊（3%）: +25コイン。宝箱（2%）: 常時コイン10枚＋未所持コレクション1種（コンプ後もコイン継続）
- ショップ（コイン）: つるはし強化・地質調査の2種（手下雇用は仕様変更で削除）
- プレステージ「埋め戻し」: 深度12以深で結晶獲得→新坑道へ。書庫・宝物・結晶・恒久ツリー・仲間は永続、コイン・部屋・ショップ強化はリセット

### UI/ウィンドウ
- **小窓（常駐ミニウィンドウ）**: 320×48、**左下**に配置（左端から約232px≈2.5cm＋微調整、下端から約12px≈0.3cm浮かせ）。右下は音量/Wi-Fiアイコンと被るため避けた
- クリックで**中央の展開ウィンドウ**（1152×648）が開く。ESCまたは「タスクバーへ戻る」で復帰
- 展開画面は右側に**ボタンパネル**（2列グリッド、44px高・16pxフォント、選択中は金色ハイライト）。掘る/宿舎/酒場/祭壇/書庫/宝物/ショップ/埋め戻し/調査書/高さ切替/言語切替/戻る/終了
- **ズームは128pxセル**（トンネルのクローズアップ、視界7×5マス程度）。カメラは**常に主人公を追尾**
- 自動掘削ボタンは「なし/あり」のシンプルなトグル（横に広げる方式はコアに残るがUIから削除）

### アート（本物のイラストが入っている）
- **主人公とリコは、ユーザー提供のキャラクターシートから実際に切り出した素材**（生成ドット絵ではない）
- 切り出しツール: `tools/extract_sprites.gd`（Godotヘッドレスツール）。シート原本は `assets/reference/*_sheet.png`
- 処理内容: 指定座標でクロップ→背景を境界フラッドフィルで透過→トリミング→正方形に整形→128pxにLanczosリサイズ
- 主人公: 3コマの掘削ループ（構え→振りかぶり→打撃）。左向きに掘るときは自動で左右反転（MINION_NATIVE_FACING）
- リコ: 歩き2コマ。「気づく/手記を読む/考え込む/驚く/励ます」の5ポーズも抽出済み（将来のイベント演出用、まだ未使用）
- 掘削エフェクトは**掘っている地形の色に応じた破片**を手続き生成（土=茶、岩=灰、水脈=青、遺構=砂色。土煙を素材に焼き込まない）
- 地形タイル5種・部屋3種・拠点旗は、まだ**生成した仮のドット絵**（本番差し替え可能）

### 画像の受け渡し方法（重要な運用ルール）
- チャットに貼られた画像はファイルとして直接取り出せない
- ユーザーに「画像をコピーして」と頼み、「コピーした」と言ってもらったら、PowerShellで
  `[System.Windows.Forms.Clipboard]::GetImage()` を使ってクリップボードから直接保存する
  （**最後にコピーした1枚のみ**保持されるので、複数キャラは1人ずつ順番に）

## 開発フロー（新セッションで踏襲すること）

1. 変更後は必ずGUTテストを実行し、全件グリーンを確認:
   ```
   & C:\src\tools\godot\Godot_v4.7-stable_win64_console.exe --headless --path C:\src\underdesk -s res://addons/gut/gut_cmdln.gd -gdir=res://tests -ginclude_subdirs -gexit
   ```
2. 新しい class_name / データファイル / PNG を追加したら再インポート必須:
   ```
   & C:\src\tools\godot\Godot_v4.7-stable_win64_console.exe --headless --path C:\src\underdesk --import
   ```
3. コミット後は `git push origin main`
4. PowerShell 5.1でgit commitする際、**メッセージに二重引用符を含めない**（here-stringでも引数が壊れる）
5. 決定性・オフライン等価性・セーブ互換の3原則を破らない（詳細はCLAUDE.md §鉄則）
6. コンテンツ追加はコードを触らずJSON/CSVで完結する設計（CLAUDE.mdのレシピ集を参照）
7. UIの位置・サイズ調整はユーザーがcm単位で指示してくる。実機ウィンドウ位置はPowerShellのWin32 API（GetWindowRect）で実測して確認する運用
